## The Vision

As someone who loves both **code** and **poetry**, I wanted to create a space where these two worlds could meet. Poetry-web2 is my attempt at building a digital sanctuary for poets.

### Why Payload CMS?

I chose Payload CMS for several reasons:

- **Flexibility**: It's headless, so I can design the frontend exactly how I want
- **Developer Experience**: Built with TypeScript and modern tools
- **Content Management**: Perfect for managing poems, authors, and collections

## Technical Stack

Here's what powers the platform:

\`\`\`typescript
const stack = {
  frontend: "Next.js 15",
  backend: "Payload CMS",

  styling: "Tailwind CSS",
  deployment: "Vercel"
}

### Key Features

1. **Rich Text Editor** - Write poems with proper formatting
2. **Collections** - Organize poems by theme or mood
3. **Author Profiles** - Showcase poet biographies
4. **Search & Filter** - Find poems by tags, authors, or keywords

## The Design Philosophy

> "Poetry is when an emotion has found its thought and the thought has found words." - Robert Frost

I wanted the design to be:

- *Minimal* yet **expressive**
- Easy to read with great typography
- Focused on the content, not distractions

## Challenges & Learnings

Building this taught me a lot about:

| Challenge | Solution |
|-----------|----------|
| Content Structure | Used flexible schemas in Payload |
| Performance | Implemented ISR for fast loading |
| SEO | Added proper metadata for each poem |
